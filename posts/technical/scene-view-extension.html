Scene View Extension 是 UE 提供的一个扩展点，可以在不修改引擎源码的情况下，往主渲染流程里插入自定义步骤。

基本结构

继承 FSceneViewExtensionBase，重写你关心的回调：

class FMyViewExtension : public FSceneViewExtensionBase {
public:
    FMyViewExtension(const FAutoRegister& AutoReg)
        : FSceneViewExtensionBase(AutoReg) {}

    virtual void SetupViewFamily(FSceneViewFamily& InViewFamily) override {}
    virtual void SetupView(FSceneViewFamily& InViewFamily, FSceneView& InView) override {}
    virtual void BeginRenderViewFamily(FSceneViewFamily& InViewFamily) override {}
    virtual void PrePostProcessPass_RenderThread(
        FRDGBuilder& GraphBuilder,
        const FSceneView& View,
        const FPostProcessingInputs& Inputs) override {}
};

注册到引擎：

FSceneViewExtensions::NewExtension<FMyViewExtension>();

可用的回调点

  • SetupViewFamily：在游戏线程，view family 创建之前
  • SetupView：每个 view 创建时
  • BeginRenderViewFamily：游戏线程，渲染开始前
  • PrePostProcessPass_RenderThread：渲染线程，post process 之前
  • PostRenderBasePass_RenderThread：渲染线程，base pass 之后
  • PreRenderView_RenderThread：每个 view 渲染前

移动端能用吗？

能用，没问题。Scene View Extension 是框架层面的，不绑渲染后端。移动端和桌面端都走得通。

性能开销主要看 hook 里干了啥。空 hook 几乎零成本，加了 pass 才会真正吃带宽。

一个简单的示例

假设我们要在 PostProcess 之前画一个 fullscreen 调试层：

virtual void PrePostProcessPass_RenderThread(
    FRDGBuilder& GraphBuilder,
    const FSceneView& View,
    const FPostProcessingInputs& Inputs)
{
    if (!View.bIsViewInfo) return;

    const FScreenPassTexture SceneColor((*Inputs.SceneTextures)->SceneColorTexture);

    // 拿 scene color 创建一个新的 render target
    FRDGTextureDesc Desc = SceneColor.Texture->Desc;
    Desc.Reset();
    Desc.Flags |= TexCreate_RenderTargetable | TexCreate_ShaderResource;

    FRDGTextureRef Output = GraphBuilder.CreateTexture(Desc, TEXT("DebugOverlay"));

    // 画一个 quad
    AddDrawScreenPass(
        GraphBuilder,
        RDG_EVENT_NAME("DebugOverlay"),
        View,
        FScreenPassTextureViewport(Output),
        FScreenPassTextureViewport(SceneColor),
        TShaderMapRef<FDebugOverlayPS>(GetGlobalShaderMap(View.GetFeatureLevel()))
    );
}
注意：拿 scene color 作为输入会强制 tile flush，移动端要谨慎。

什么时候不要用

Scene View Extension 适合插入逻辑步骤，但不适合做"每天都要画"的 UI。如果你的功能是持续的 UI 层，最好走 UMG 或 Material 系统。
