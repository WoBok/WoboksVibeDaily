今天想解决一个 UE 渲染问题：让一个全景球可以遮挡所有物体，但又允许某些特定物体在它之上显示。

问题

全景球做成透明 + 不写深度测试（DisableDepthTest），可以"挡住"所有物体，因为它始终画在最上面。然后想让部分物体在它之上——这些物体如果也用透明，调整透明层级可以在最前面，但透明物体之间没有深度关系，会出现自身乱序的问题。

所以目标是：让部分物体在所有透明物体渲染完成之后才渲染。

几个入口

1. Scene View Extension

用 FSceneViewExtensionBase 加 hook，挂到 PrePostProcessPass 或之后的 PostRenderBasePass。可以自己控制 RenderState，确保画在最上面。移动端和桌面端都能用，性能开销主要看 hook 里干了啥。

2. Translucency Pass

在 FMeshPassProcessor 里把这类物体分到 ETranslucencyPass::AfterDOF 或自定义的 pass，让它在所有标准透明之后画。不开深度测试，让它自然挡在前面。

3. 材质层面

材质里 TranslucencyPass 选特定 pass（如 After DOF），配合 DisableDepthTest。效果和我们想要的完全一致。

移动端性能问题

移动端是 tile-based 架构，GPU 和 CPU 共享系统内存（UMA），带宽是共享的，tile memory 也有限。多加一个 full-screen pass 意味着：

  • 多一次 RenderTarget 切换 → tile flush，framebuffer 要 dump 到主存
  • 如果 pass 还要读前面的 scene color，那次 load 就是带宽炸弹
  • MSAA 的 resolve 也是隐性开销

硬插一个 pass 进去，等于打断 UE 移动端的合并优化路径。

更轻量的做法

其实不用额外 pass。换个思路：

  1. 材质里直接配 AfterDOF pass + DisableDepthTest：走 UE 自己的 Translucency 流程，自动在 mobile 优化路径里，不需要新建 RenderPass。
  2. CustomDepth + 自定义 MeshDrawCommand：让某些物体在 Translucency 之后单独画一次，render target 复用现有的。
  3. 最小的方案：直接用 ETranslucencyPass::AfterDOF + DisableDepthTest=true，绝大部分设备都支持，带宽基本没增加。

最后我选了方案 1——材质层配置，简单、稳定、不破坏 mobile 优化路径。
