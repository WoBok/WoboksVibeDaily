dumbbear（和它的兄弟 dumbbear2）是我在 wobok.tech 下跑的两个短链服务。功能很简单——把长 URL 变成短 URL。

架构

  • 前端：纯静态 HTML
  • 后端：没有后端，纯 nginx rewrite
  • 存储：JSON 文件，存映射关系

为什么不写后端

短链这个场景，90% 是读、10% 是写。读的部分可以直接由 nginx 处理——匹配短路径，返回 302 重定向到长 URL。写的部分很少，可以手动维护 JSON。

这样省了一个进程、一份部署、一个出 bug 的地方。

配置

location ~ ^/d/([a-zA-Z0-9]+)$ {
    # 实际生产中用 lua 查 json
    # 这里简化：用 try_files + 预生成
    try_files /d/$1.json =404;
}

# 或者直接 rewrite
location ~ ^/d/([a-zA-Z0-9]+)$ {
    return 302 https://example.com/long/url/$1;
}

怎么加新链接

手动加 return 302 那一行，然后 nginx -s reload。

听起来很土，但一个短链服务一年也加不了 50 个链接，这个效率比写后端高。

不是所有问题都需要后端。最简单的方案，往往是最耐用的。
