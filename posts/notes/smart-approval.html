Hermes 有三种 approval 模式：manual、smart、off。

各模式的行为

  • manual：每次执行命令都弹窗问
  • smart：常规命令自动过，危险命令弹窗
  • off：全部自动执行

smart 模式下：ls、cat、grep、systemctl status、pip install 自动过；rm -rf、git push -f、curl POST、ssh 弹窗。

怎么改

改 ~/.hermes/config.yaml：

approvals:
  mode: smart

改完需要重启 gateway 才生效：

systemctl --user restart hermes-gateway

重启的小坑

这条命令必须在 gateway 外面跑——比如 SSH 进去或者另一个终端。如果在 gateway 内部进程里执行，会被自残保护拦住。

什么算危险

smart 模式的"危险"判定主要看这几类：

  • 删除：rm -rf、find -delete
  • 版本控制强制操作：git push -f、git reset --hard
  • 对外发消息：curl POST、mail、hermes send
  • 系统重启：systemctl restart、reboot
  • 不可逆操作：dd、mkfs
如果觉得太松 → off；太紧 → manual。smart 是大多数人的甜蜜点。
