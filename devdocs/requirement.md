主要逻辑：
    文章以文件的方式存放于各个文件夹，文件夹是分类目录，每个分类（叶子文件夹）中有若干文章，主界面左侧目录加载文件夹树，右侧文章列表显示当前分类（叶子文件夹）中的文章文件，文章列表项包含创建日期、文件标题和文章总结，点击文章列表项，打开文章，跳转到文章页面

界面风格：
    Botanical Organic Serif风格

文件格式：
    1. Markdown，需要支持完整的Markdown文档解析，包含Latex，代码（支持各种语言）等
    2. HTML

文件夹逻辑示例：
    Folder : 0x0 - Inbox（分类目录，目录下再无目录，也为叶子目录）
        File : Note1.md
        File : Note2.html
    Folder : 0x1 - Concepts（分类目录）
        Folder : 0x0 - Math（分类目录，目录下再无目录，也为叶子目录）
            File : Note1.md
            File : Note2.html
        Folder : 0x1 - Art（分类目录，空目录）

    只有叶子目录可以存放文件

主界面：
    1. 左侧为目录，保持当前样式，目录名称左侧的箭头改成墨绿色圆点，选中的目录及所有父目录亮起浅绿色，颜色都是莫兰蒂低饱和，当点击目录时，若有子目录将展开目录，若为叶子目录则加载当前目录下的文章到右侧，空目录点击无反应，再次点击目录则收起当前目录，即便是有子目录打开，名称左侧圆点恢复墨绿，每个目录上显示当前文件夹下包含子目录一共有多少文章的数量数字，保持当前样式即可
    2. 右侧为当前目录下的文章列表，保持当前样式，列表项中包含的信息有时间，标题和总结，列表项的高度随总结字数变化，总结内容要可以完全显示，当前有一个阅读时间，去掉这个功能
    3. 默认打开主页时，右侧文章列表现在显示Posts，改成Latest，这里的显示逻辑是按照时间顺序显示所有笔记
    4. 主界面标题栏右侧的日期去掉

文章页面：
    加载逻辑：
        1. 若文件格式为Markdown，则解析并渲染Markdown文件，文章页面元素自上往下，依次是1. 目录路径 · 时间 2. 文章标题（文件名称） 3. Markdown文件，其实和当前一致，只不过当前加载的时HTML，用了CSS样式，现在的HTML的文章方式去掉，直接解析md即可，根据当前界面风格对md做样式优化即可
        2. 若文件格式为HTML则直接加载HTML文件，不需要做任何样式改变，原HTML什么样式，保留即可，两套加载逻辑不要有耦合，是分开独立的
    页面布局及功能：
        1. 左侧为文章目录大纲，贴左侧边缘显示，默认情况下根据标题数量显示长短不一的线条，长短粗细根据标题的级别不同，注意要美观，当鼠标移入则显示完整文字大纲，要有动画效果，鼠标指向的标题进行放大，上下邻近的也要有一定放大效果，类似于Mac的Dock栏的缩放效果，移出时则完整文字大纲收起变为线条，也要有动画，无论是线条模式还是完整文字大纲模式都要在页面垂直方向居中显示，而且还是以Overlay（浮在当前页面上）方式存在，不影响中间文章的布局
            a. 若文章格式为Markdown，则显示#语法的标题
            b. 若文章格式为HTML，则显示<h>标签语法的标题
        2. 中间加载的为Markdown或HTML的文章，其实是占据这个页面
        3. 右侧为文章中的引用，贴右侧边缘显示逻辑与文章目录大纲一致
        4. 在标题栏右侧（当前日期的显示位置）添加返回按钮，原来的返回按钮移除，返回按钮文字为英文
    文章引用功能说明：
        1. Markdown：自动加载Markdown中的[显示文本](链接地址)这种语法，在文章内显示文本处加上脚注，右侧文章引用显示时与左侧文章目录大纲字体颜色一致，条目内容是显示文本，有序号，序号与脚注对应，且为超链接，点击可跳转，不要有下划线等，样式与左侧文章目录大纲保持一致即可
        2. HTML： 自动加载<a href="链接地址">显示文本</a>这种语法，显示逻辑同上

manifest构建逻辑：
    manifest分类：
        1. 根目录（当前为posts文件夹）下有一个manifest文件，记录所有文章列表，包含创建时间、标题和总结
        2. 每个叶子文件夹目录下有一个manifest文件，记录当前文章列表，包含创建时间、标题和总结
    叶子文件夹文章列表：
        当用户点击左侧列表项叶子文件夹时，读取当前文件夹下的manifest文件，若没有则创建，若有则对比manifest第一行的标记，此标记代表本文件内缓存的文章数量，若此标记与当前文件夹下的html+md的数量一致，则返回文章列表，若不一致，则重新构建，并更新标记
    默认打开主界面全部文章列表：
        1. 网站打开时读取根目录（当前为posts文件夹）下的manifest文件，如果没有则创建，此时执行整个文件夹的扫描，全部构建，若有则直接返回
        2. 当叶子文件夹发现文章数量不一致导致重新构建叶子文件夹的manifest时，则同步执行对根目录manifest的增量更新
        3. 根目录文章列表manifest一定是按照时间顺序排列
    使用技术：
        因为不想每次发布文章都构建一次manifest文件，因为我可能频繁发布，所以希望带后端，直接使用Node.js做判断
